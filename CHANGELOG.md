## 版本说明

### 版本号规则

采用 [语义化版本](https://semver.org/lang/zh-CN/) 规则：

- **主版本号**: 不兼容的 API 修改
- **次版本号**: 向下兼容的功能性新增
- **修订版本号**: 向下兼容的问题修正

### 变更类型说明

- 🆕 **新增**: 新功能
- ⚡ **改进**: 现有功能优化
- 🐛 **修复**: 错误修复
- 📚 **文档**: 文档更新
- 🔄 **重构**: 代码重构
- 🎉 **里程碑**: 重要版本发布
- 📈 **性能**: 性能优化
- 🛡️ **安全**: 安全修复
- 🗑️ **移除**: 功能移除

### 发布周期

- **主版本**: 根据重大功能更新发布
- **次版本**: 每月发布一次，包含新功能和改进
- **修订版本**: 根据需要发布，主要修复关键问题

### 兼容性说明

- 向后兼容性保证：同一主版本内的次版本和修订版本都保持向后兼容
- 配置文件兼容：新版本会自动适配旧版本配置
- 数据格式兼容：数据文件格式保持向后兼容，如有变更会提供迁移工具

### 升级指南

#### 从 1.1.x 升级到 1.2.0

1. **备份配置**：备份现有的配置文件
2. **更新代码**：拉取最新代码
3. **安装依赖**：更新依赖包
4. **配置迁移**：新配置项会使用默认值，可根据需要调整
5. **测试验证**：运行回测验证功能正常

#### 配置变更

新增配置项：

```yaml
# 策略信号去重
strategy:
  signal_deduplication:
    enabled: true
    same_day_filter: true
    same_week_filter: false

# 批量回测
batch:
  parallel_workers: 4
  chunk_size: 10
  output_format: "csv"
```

## 贡献指南

如果您想为项目做出贡献：

1. Fork 项目仓库
2. 创建功能分支
3. 提交代码更改
4. 创建 Pull Request
5. 等待代码审查和合并

### 提交信息规范

请使用以下格式的提交信息：

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

类型包括：
- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

示例：
```
feat(strategy): add RSI indicator support

Add RSI technical indicator with configurable period.
Include overbought/oversold signal generation.

Closes #123
```
